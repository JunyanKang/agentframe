# Documentation

- `.codex/README.md`.

## Documentation Rules
- Keep reusable docs generic.
- Mark unknown facts explicitly.
- Do not document nonexistent product behavior.
- Update docs when workflows or contracts change.

## Required Documentation Updates
- Update changelog for releases.
- Update framework version for reusable contract changes.
- Update project memory for meaningful state changes.

## Documentation Gaps

## Examples Policy
Examples must be neutral and must not invent a domain-specific project.
